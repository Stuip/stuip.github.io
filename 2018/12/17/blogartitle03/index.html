Cookie 和 Session 小结

发表于 2018-12-17 | 分类于 HTTP 协议

Cookie 机制

        Cookie 技术是客户端的解决方案，Cookie 是服务器发给客户端的特殊文本，而这些特殊信息以文本文件的方式储存在客户端，
每次客户端给服务端发送请求都会带上这些特殊信息。

具体流程：

  1. 当用户使用浏览器访问一个支持 Cookie 的网站的时候，用户会提供包括用户名在内的个人信息并且提交至服务器

  2. 服务器在向客户端回传相应的超文本的同时也会发回这些个人信息（储存在 Response header中）,浏览器会将这些特殊信息储存在本机的统一位置

  3. 如果浏览器再次向服务器发送请求的时候，Cookie 会将储存在 Request Header中发到服务器。

  4. 服务器接收到浏览器的请求，就能够通过分析存放于请求头的 Cookie 得到客户端特有的信息，从而动态生成与该客户端相对应的内容。

问题

        在程序中，一个用户所有的操作都属于同一个会话，而 web 应用程序使用 HTTP协议传输数据，但是HTTP协议是无状态的协议，一旦数据交换完毕，
客户端与服务器端的连接就会关闭，再次交换数据需要建立新的连接，那么怎么判断是否当前用户是否是刚刚那个会话呢？

这时候就需要用 Cookie 机制来实现会话追踪，在每次请求中，Header（Request-Header）有两个属性：Set-Cookie 以及 Cookie 分配是设置 Cookie 和发送 Cookie

Set-Cookie

当访问一个带有的 Cookie 的网站的时候，服务器会返回客户端一个 Http 响应信息（Response），其中头部就包含 Set-Cookie，
这就说明了服务器给请求建立了一个 Cookie，并且在以后的请求中都会带有这个 Cookie 到服务器，直至到 Cookie 过期。

Cookie

Cookie 有一个生命周期

  • 如果生命周期是整个会话的话，那么浏览器就会将 Cookie 存在内存中，只要浏览器关闭就会自动清除内存的 Cookie
  • 如果生命周期是永久的话，那么浏览器就会将 Cookie存在硬盘中，但是浏览器关闭也不会将 Cookie 清除

Cookie 的一些属性

  • Cookie 名称：Cookie 名称一旦创建，名称不能修改。
  • Cookie 值：
    • 如果值为 Unicode 字符，则需要字符编码
    • 如果值为二进制数据，则需要使用 BASE64 编码。

  • Cookie 失效时间／过期时间： maxAge/Expires，单位为秒，默认为-1

    • 如果值为正数，说明时间大于这个值之后，Cookie 失效
    • 如果值为负数，说明该 Cookie 为临时的，关闭浏览器即失效，浏览器也不会以任何形式保存
    • 如果值为零，说明删除该 Cookie

  • Cookie作用域：
            Domain 和 Path 标识定义了Cookie的作用域：即Cookie应该发送给哪些URL。
    Domain 标识指定了哪些主机可以接受Cookie。如果不指定，默认为当前文档的主机（不包含子域名）。如果指定了Domain，则一般包含子域名。

##Session 机制

        Session则是服务端的解决方案，它是通过服务器来保持状态的。客户端访问服务器时候，服务器把客户端信息以Session的形式记录在
服务器上，Session会在服务器给客户端所开辟的存储空间，在其中保存的信息就是用于保持状态，而服务器给客户端创建Session的同时，
也会生成唯一的Session ID来确保在以后的请求中来重新获得这个Session。

Session 生命周期

        Session是储存在服务器端，而服务器一般会Session存在内存中，
Session生成后，只要用户继续访问，服务器就会更新Session的最后访问时间，并维护该Session。
无论是否读写Session，服务器都认为该用户的Session“活跃（active）”了一次。

Session的有效期

       服务器会将长期没有活跃的Session从内存中删除，这个就是Session超时时间。

Session 与 Cookie之间的区别

  • Cookie是存放在浏览器上的，而Session存放在服务器上的
  • Cookie不是很安全，因为Cookie存放在客户端中，可以浏览、修改以及复制到Cookie内容
  • Session会在一定时间内保存在服务器上。当访问增多，会比较占用你服务器的性能。考虑到减轻服务器性能方面，应当使用Cookie
  • Cookie需要客户端浏览器的支持，如果客户端禁用Cookie，或者不支持Cookie，则跟踪会话失效，而不支持的Cookie的浏览器可以
    使用Session+URL重写来实现，注意的是：一切的用到Session程序的URL都要进行URL地址重写，否则Session会话跟踪还会失效。
