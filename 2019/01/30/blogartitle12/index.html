Object 源码分析

发表于 2019-01-30 | 分类于 JDK源码分析

今天学习Java继承的时候，学习到了Object是所有的类的根类，不知这个Object类是何方神圣，去看了源码

  • 我们应该了解静态代码块，静态代码块是随着类的加载而执行的，并且仅加载一次
  • 了解一下，native关键字
    • Native关键字是对于本地方法的声明，native关键字是与C++联合开发的时候的，一般是用C／C++语言操作的，例如可能需要操作内存
    • 那么Java怎么调用哪些不是原生态方法？
      • 是通过JNI(Java Native Interface)，它也是Java JDK的一部分，是Java调用其他的语言的接口
    • registerNatives： 对本地方法的注册，也就是初始化Java原生方法映射到对应的其他的语言描述方法，比如C语言的方法
  • 此方法是返回Object的运行时类,并且这个类不能被重写。

  • Java反射可以通过getClass()获得，但几乎不常用，

  • equal方法是比较俩个对象是否相等的主要方法

    • 一般不重写equal方法，那么它就相当于==，比较的是基本类型的内容和引用类型的地址，
      其实无论基本类型还是引用类型，都是比较栈内存中内存的值（String类重写了这个方法）

  • 一般针对不同的对象返回不同的对象返回不同的整数（一般通过将该对象的内部地址换成一个整数来实现）

  • 返回一个克隆对象，二者具有相同的属性，二者在堆内存中有着不同的地址

  • 返回一个对象的信息，例如： 类名@对象地址

  • notify()方法唤醒一个这个对象的监视器上等待的单个线程，被唤醒的线程直到当前的释放锁后才可以执行，并且和其他线程线程竞争对象的拥有权（没有优先级）

  • 唤醒的线程不是得到CPU的使用权，而是和其他的就绪线程竞争，也就是将阻塞状态转换为就绪状态，并不是立即执行。

  • 顾名思义，就是将这个对象上的所有的线程都就绪，并不是立即执行

  1. 使用wait(long timeout) 是当前线程等待另一个线程，直到另外一个线程调用改对象的notify()或notifyall()方法，或者等待时间已到
  • 会让当前的线程进入当前对象的等待集中，让出当前对象的监视器，直到如下几种情况才结束：
    1. 其他的线程带哦用当前对象的notify()的方法，并且该线程恰好被唤醒
    2. 其他的线程调用当前的对象的notifyall()方法
    3. 其他线程暂停该线程
    4. 等待时间已经到了

  1. 使用 wait(long timeout, int nanos)就是将nanos提高更高的精度的控制

  2. wait() 表示无限期的等待

  • 垃圾回收器觉得该对象没有被其他的任何线程调用的时候，就有该对象的垃圾回收器调用此方法
