Inerface Inheritance

由于最近在观看CS61B的Java基础部分，对于Java的继承又有新的感悟，对此做一些总结。
当使用接口继承(inerface inheritance), 需要在子类中使用implements关键词实现。

List类与ArrayList是属于”is-a”的关系,对于子类重写的方法上面加上@overraide标签。
如果在加上@overraide标签情况下，错误输入重写的方法名称，编译器会阻止编译。

Default Methods

interface中允许有自己的Default method,实现这种方法只要是在interface实现，但是这个方法里面不能包含instance variables。default method可以在subClass中使用，也可以在SubClass中重写default method。

Override VS Overload

在class中的方法可以有相同的方法名，但是有不同的参数。eg:在Math class中有add(int a, int b)和add(double a, double b)
编译器会通过传入的参数类型来判断所调用的具体方法，这就是overload方法。
而override方法需要相同的方法名、相同的返回参数以及相同的方法参数类型。

Dynamic method selection

static variable and dynamic variable

每一个Java变量都有static type，这个static type是在变量申明时候所指定的,并且是在compile-time检查。而dynamic type是在实例化时候所指定，并且在runtime检查

Casting

Casting是指在compile-time允许编译器忽略强制将Superclass转换Subclass，并调用子类的方法。

Note :Casting并没有改变变量的static/dynamic type，而是暂时将该变量装入某一指定类型中。(欺骗compiler,让它相信该变量为指定的类型, 可能会在runtime报错)

对于fifi和frou变量的总结：

variable static type dynamic type
fifi Poodle Poodle
frou Dog Poodle
  1. 第一条语句：实例化一个Poodle类型并指向fifi变量
  2. 第二条语句：
    • compile-time: 检查fifi的static type，并且检查static type类型是否有相应的wash函数.
    • run-time:调用fifi的dynamic type相应的方法, fifi调用的wash函数参数是Soap类型，但是Poodl类中的wash函数的参数为PoolSoap，但是Poodle类继承了Dog类，所以需要到Dog寻找参数为Soap类型的wash函数 => Dog is clean!
  3. 第三条语句：
    • run-time: 由于调用的wash的参数类型为PoodleSoap类型，并且在Poodle类中有相匹配的函数。 => Poodle is clean!
  4. 第四句语句：实例化一个Poodle类型指向frou变量。
  5. 第五条语句：
    • compile-time: 检查frou的类型为Dog, 并且检查是否有wash函数，如果没有此函数，则会报compile error,否则选定该函数。
    • run-time:检查frou的dynamic type中的函数是否有匹配的函数，最终只能确定在Dog类中 => Dog is clean!
  6. 第六条语句：
    • run-time:确定是否static type所选择的函数是否可行，如果可行的话，就调用，否则寻找使用dynamic type的wash函数

Summary:

  1. compiler是一个严格的检查器，它仅仅知道static type
  2. Methods Producure:
    • 在complie-time,查看static type的函数是否存在
    • 记下与之最为匹配的函数名和函数参数
    • 在runtime下，如果dynamic type中有static type中的函数（override）,则运行dynamic type中的重写的函数。
  3. Dynamic Method seletion Algorithm:
    • Goal: determine what code runs when we write x.MethodName(Y)
    • At complie time:
      1. Look at the static type / complie-time type of x
      2. Find all methods that accept the complie-time type of y;
      3. choose the most specific method and remenber its signature;
    • At runtime: Starting from the runtime-type of x, select lowest method in the inherance hierarchy that has the exact signature chosen at complie-time in steps.
